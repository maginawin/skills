# 规划项目需求

Created At: Jan 9, 2026

当识别到用户说"规划需求："或"规划："时，执行以下步骤：

## 1. 解析需求内容

提取"规划需求："或"规划："后面的内容作为需求描述。

## 2. 确认不明确的问题

如果需求中有不明确或模糊的地方，**必须先与用户确认**，再开始规划。常见需要确认的问题：
- 功能范围边界
- 技术选型偏好
- 优先级要求
- 依赖或约束条件

## 3. 分析当前项目现状

- 了解项目的技术栈和架构
- 查看现有代码结构
- 评估与新需求相关的现有功能

## 4. 制定开发计划

仔细思考，认真分析，规划内容应包括：

### 4.1 需求概述
简要描述需求的核心目标

### 4.2 复杂度评估
预测开发复杂程度（低/中/高），说明评估依据

### 4.3 分阶段实现计划
将开发任务分成不同阶段，每个阶段包括：
- 阶段目标
- 具体任务清单
- 预期产出

### 4.4 风险与注意事项
列出可能的技术风险和需要注意的问题

## 5. 保存计划文件

将规划文档保存到 `docs/` 目录下，文件名格式：

```
yyMMdd_HHmm_规划_功能简洁说明.md
```

示例：`251204_1620_规划_设备控制.md`

## 注意事项

- 规划前必须确认所有不明确的问题
- 保持计划的可执行性和清晰度
- 复杂度评估要有依据
- 阶段划分要合理，便于迭代开发
