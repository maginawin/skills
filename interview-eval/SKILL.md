# 面试评估与规划

分析候选人简历，结合岗位需求文档，生成包含优劣势分析、岗位匹配度评估、面试题及参考答案的 Word 文档。

## 输入要求

需要用户提供以下内容（通过文件路径或对话中指定）：

1. **候选人简历**（PDF 或 Word）— 必须
2. **岗位需求/人力计划文档**（Word/PDF）— 如有则读取，没有则根据用户描述的岗位要求进行分析
3. **侧重方向**（可选）— 如用户补充了特殊培养方向（如"侧重固件测试"），在分析中体现此偏差

## 执行流程

### 第一步：读取文档

- 读取候选人简历，提取个人信息、教育背景、工作经历、技能清单、项目经验
- 读取岗位需求文档（如有），提取岗位要求、技能要求、职责描述
- 如果文档是 PDF，使用 pdfminer 提取文本；如果是 docx，使用 python-docx 提取
- 提取失败时告知用户并尝试其他方式

### 第二步：候选人分析

#### 基本信息表
整理为结构化表格：姓名、年龄、学历、工作年限、最近公司、求职意向。

#### 优势分析
从以下维度挖掘候选人亮点（通常 4-6 项）：
- 与目标岗位直接相关的行业/项目经验
- 技术栈匹配度
- 跨领域/跨平台经验
- 管理/带人经验
- 工具链掌握程度

每项优势给出标题和具体支撑论据，论据要引用简历中的具体内容。

#### 劣势分析
客观评估候选人短板（通常 3-5 项）：
- 学历/专业背景
- 目标方向的技术深度不足
- 自动化/工程化能力缺口
- 离职原因/职业稳定性风险
- 经验含金量问题

#### 岗位匹配度评估
用表格逐维度评分（★ 制，满分 5 星），维度根据岗位动态选择，通常包括：
- 学历、核心经验年限、主要技能匹配、行业经验、培养潜力等
- 如有侧重方向，增加对应的潜力评估维度
- 给出综合匹配度百分比和是否建议进入面试的结论

### 第三步：面试流程规划

面试分为两轮，总时长约 90-120 分钟。

#### 第一轮：技术面试（60-70 分钟）

分为四个 Part，每个 Part 约 15 分钟：

**Part 1 — 项目经验深挖**
- 3 道题，围绕简历中最相关的项目经历设计
- 考察项目参与深度、技术方案理解、实际贡献

**Part 2 — 技能专项**
- 3 道题，针对简历中列出的核心技能设计
- 前 2 题匹配其当前水平，第 3 题为超纲题（标注"超纲"）
- 超纲题用于测试技能掌握的深度和边界

**Part 3 — 方法论与工具**
- 3 道题，考察工作方法、工具使用、问题排查思路
- 前 2 题为实操场景题，第 3 题为超纲的架构/方案设计题

**Part 4 — 目标方向专项**（如有侧重方向）
- 3-4 道进阶/超纲题，评估候选人向目标方向转型的潜力
- 题目围绕目标方向的基础概念和实际场景设计
- 候选人不需要全部答对，重点看思维方式和学习意愿

如果没有特殊侧重方向，Part 4 可替换为"综合技术题"，考察更广泛的技术视野。

#### 第二轮：综合面试（30-40 分钟）

- 3-4 道题，考察离职原因、职业规划、转型意愿、沟通能力、管理潜力等
- 每道题标注考察点而非标准答案

#### 面试题要求
- 每道题都要有**参考答案**（技术题）或**考察点**（综合题）
- 参考答案要详细，列出关键得分点，面试官可据此快速评判
- 超纲题的参考答案需注明"能答出 XX 为加分项"

### 第四步：评分建议表

生成评分维度表格，包含：
- 评估维度（根据岗位动态确定，通常 4-6 个维度）
- 权重（百分比，总和 100%，根据岗位侧重动态分配）
- 评分标准（对应哪些面试题）

给出录用建议阈值：
- ≥ 70 分：建议录用
- 60-69 分：条件录用（明确培养计划）
- < 60 分：不建议录用

### 第五步：总结与建议

- 候选人整体画像（2-3 句话概括）
- 面试重点关注事项（4 项左右的 bullet points）

### 第六步：生成 Word 文档

使用 `scripts/generate_docx.py` 脚本生成格式化的 Word 文档。

将分析结果整理为 JSON 数据，然后调用脚本：

```bash
python3 ~/.claude/skills/interview-eval/scripts/generate_docx.py \
  --data /tmp/interview_eval_data.json \
  --output "<docs目录>/候选人姓名_岗位名称_面试评估与规划.docx"
```

JSON 数据结构见 `scripts/generate_docx.py` 中的注释。

如果脚本执行失败，回退到在 Python 中直接使用 python-docx 生成文档。

## 输出

- Word 文档保存到 `docs/` 目录，文件名格式：`{姓名}_{岗位名称}_面试评估与规划.docx`
- 告知用户文档已生成及保存路径
